## 📋 版本记录格式

```

```

---

## 🏷️ 提交类型说明

- **[INIT]** - 项目初始化
- **[FEAT]** - 新功能开发

- **[PERF]** - 性能优化

## 📝 使用说明

### 如何记录新版本

1. **确定版本号**: 根据语义化版本规范(主版本.次版本.修订号)
2. **选择提交类型**: 从上述提交类型中选择最合适的
3. **填写更新内容**: 详细列出本次更新的所有内容
4. **记录Git命令**: 记录实际使用的Git命令
5. **添加备注**: 简要说明本次更新的背景或重要性

### 版本号规范

- **主版本号(X.0.0)**: 重大功能更新或架构变更
- **次版本号(0.X.0)**: 新功能添加
- **修订号(0.0.X)**: Bug修复或小改进
